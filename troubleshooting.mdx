# Troubleshooting

The most common Headmaster issues, with the shortest fix that works.

---

## Common issues

<AccordionGroup>
  <Accordion title='"Runtime not connected" banner'>
    The runtime is the background service. When it stops responding, the app shows a banner at the top of the chat.

    **Quick fix:**
    1. Open **Settings → Headmaster's Library → Runtime → Restart**.
    2. Wait five seconds.
    3. The status indicator should flip back to Connected.

    **If the restart doesn't help:**
    1. Open the log file at `<data folder>/logs/<today>.log`.
    2. Look for the last error before the banner appeared. Most common causes: missing provider credentials, a port conflict, or a runtime that crashed on startup.
    3. If you see a port conflict, close any other Headmaster windows and any other app that might be holding the port.

    **Last resort:** Quit Headmaster fully (on macOS, `Cmd+Q`), wait ten seconds, then reopen.
  </Accordion>

  <Accordion title='"401 Unauthorized" on Memory, Skills, or Runtime tabs'>
    A 401 means the app and the runtime have lost their shared session token. This usually happens after a runtime restart the app didn't see.

    **Quick fix:**
    1. Open **Settings → Headmaster's Library → Runtime → Restart**.
    2. The new runtime issues a fresh token.
    3. Reload the page you were on.

    **If 401s keep coming back:** The runtime is restarting in a loop. Open the log and look for the error before each restart. Most common cause: a malformed `config.yaml` from a manual edit. Fix the config and restart.
  </Accordion>

  <Accordion title="Runtime crash-restart loop">
    If the runtime exits immediately after starting (you see "runtime exited code=1" repeatedly in the logs):

    1. Check the log file for the last error before the exit.
    2. Common causes:
       - **Missing dependency** — on Linux, a missing `libgtk-3` or `libnss3` package.
       - **Corrupted config** — a syntax error in `config.yaml`.
       - **Port conflict** — another process is holding the port Headmaster needs.
       - **Permission issue** — the data folder is not writable by the current user.
    3. Fix the underlying issue, then restart Headmaster.
  </Accordion>

  <Accordion title='"Check for updates" returns nothing'>
    The updater talks to the Headmaster release endpoint. If it can't reach it, you see an error in the update modal.

    **Quick fix:**
    1. Check your network connection.
    2. On a corporate network, the release endpoint may be blocked. Ask your network admin to allow `gcaplabs.com`.

    **Manual update:** Download the latest installer from the Headmaster site and run it over the existing install. Your data is preserved.
  </Accordion>

  <Accordion title='"Update available" but won't install'>
    The installer needs write access to the install location. If you installed as an admin user and are now running as a non-admin user, the install will fail.

    **Fix:** Run Headmaster as the same user that installed it, or reinstall in portable mode and copy your data folder over.
  </Accordion>

  <Accordion title="A chat won't load">
    If a specific conversation won't open but others do:

    - **Interrupted session** — the session may have been interrupted mid-stream. Click the conversation, wait for the recover prompt, and click **Recover**.
    - **Corrupted transcript** — open **Settings → Headmaster's Library → Runtime → Sessions**, find the session ID, and delete the local cache. The conversation will reload from the runtime's copy.
    - **Missing model** — if the model used in that conversation is no longer available (provider key revoked, model deprecated), add the provider back or switch the conversation's model.
  </Accordion>

  <Accordion title="White screen on launch">
    If the main window shows a blank white screen:

    1. Check the log file for renderer errors.
    2. Most common cause: corrupted renderer cache. Delete it:
       - **Windows:** `%APPDATA%\Headmaster\cache\renderer`
       - **macOS:** `~/Library/Application Support/Headmaster/cache/renderer`
       - **Linux:** `~/.config/Headmaster/cache/renderer`
    3. Restart Headmaster.
  </Accordion>

  <Accordion title="Port conflict">
    If another application is using the port Headmaster needs:

    1. Open the log file and look for "port already in use" or "EADDRINUSE".
    2. Find the process holding the port:
       - **Windows:** `netstat -ano | findstr :<port>`
       - **macOS/Linux:** `lsof -i :<port>`
    3. Stop the conflicting process or configure it to use a different port.
    4. Restart Headmaster.
  </Accordion>

  <Accordion title="Runtime won't start on Linux">
    Common Linux-specific issues:

    - **Missing libraries** — install `libgtk-3-0`, `libnss3`, `libasound2`, `libxss1`.
    - **Permission denied** — ensure the data folder is owned by the current user: `sudo chown -R $USER:$USER ~/.config/Headmaster`.
    - **AppImage won't run** — ensure it's executable: `chmod +x Headmaster-*.AppImage`.
    - **Running as root** — add the `--no-sandbox` flag.
  </Accordion>
</AccordionGroup>

## Collecting logs for bug reports

When reporting a bug, include:

1. The most recent log file from `<data folder>/logs/`.
2. The Headmaster version (from **Settings → About**).
3. Your OS and version.
4. Steps to reproduce the issue.

Use **Settings → About → Report a bug** to send this automatically — it attaches the log file, app version, OS info, and the last 5 conversation IDs (not content — just IDs for correlation).
