# Channels

Channels are the messaging platforms the agent can be reached on. With a channel connected, you can send Headmaster a message from your phone and the agent's reply lands in the same conversation history.

---

## Supported platforms

| Channel | Type | Setup difficulty |
|---|---|---|
| **Telegram** | Bot API | Easy |
| **WeChat Work** | Enterprise API | Intermediate |
| **DingTalk** | Bot Webhook | Intermediate |
| **Lark** | Bot API | Intermediate |
| **WeCom** | Enterprise API | Intermediate |
| **iMessage** | Apple Messages bridge | Advanced |
| **WhatsApp Business** | Cloud API | Intermediate |

---

## Platform capabilities

| Channel | File send | File receive | Inline buttons |
|---|---|---|---|
| Telegram | Yes | Yes | Yes |
| WeChat Work | Yes | Yes | No |
| DingTalk | Yes | Limited | No |
| Lark | Yes | Yes | Yes |
| WeCom | Yes | Limited | No |
| iMessage | No | No | No |
| WhatsApp Business | Yes | Limited | Yes |

---

## Connecting a channel

<Steps>
  <Step title="Open Channels settings">
    Go to **Settings → Headmaster's Library → Channels**.
  </Step>
  <Step title="Pick a channel and connect">
    Click the channel and click **Connect**. Follow the platform-specific flow below.
  </Step>
  <Step title="Test">
    Send a message from your phone to the channel to confirm it's working.
  </Step>
</Steps>

## Setup guides

<Tabs>
  <Tab title="Telegram">
    <Steps>
      <Step title="Create a bot">
        Message [@BotFather](https://t.me/BotFather) on Telegram. Send `/newbot` and follow the prompts to name it. Copy the bot token BotFather gives you.
      </Step>
      <Step title="Connect in Headmaster">
        Go to **Channels → Telegram → Connect**. Paste the bot token and save.
      </Step>
      <Step title="Test">
        Send a message to your bot on Telegram.
      </Step>
    </Steps>

    **What works:**
    - Text messages (send and receive)
    - File attachments (send and receive)
    - Inline buttons for approvals and clarifications
    - Proactive notifications from scheduled tasks
    - Group chats (the bot responds to mentions)
  </Tab>

  <Tab title="DingTalk">
    <Steps>
      <Step title="Create a robot">
        Log in to the [DingTalk developer console](https://open-dev.dingtalk.com/). Create a custom robot: go to your group → Group Settings → Group Assistant → Add Robot → Custom. Note the **Webhook URL** and **Secret** (if using sign mode).
      </Step>
      <Step title="Connect in Headmaster">
        Go to **Channels → DingTalk → Connect**. Enter the Webhook URL and Secret. Save.
      </Step>
    </Steps>

    **What works:**
    - Text messages (send and receive)
    - File attachments (limited)
    - Text-based approvals (DingTalk doesn't support inline buttons)
    - Proactive notifications from scheduled tasks
  </Tab>

  <Tab title="WeChat Work">
    <Steps>
      <Step title="Create a custom application">
        Log in to the [WeChat Work admin console](https://work.weixin.qq.com/). Go to Application Management → Self-built → Create. Note the **Corp ID**, **Agent ID**, and **Secret**.
      </Step>
      <Step title="Configure receive message URL">
        Set the callback URL to Headmaster's webhook endpoint. Set the callback token and encoding key.
      </Step>
      <Step title="Connect in Headmaster">
        Go to **Channels → WeChat Work → Connect**. Enter the Corp ID, Agent ID, and Secret. Save.
      </Step>
    </Steps>

    **What works:**
    - Text messages (send and receive)
    - File attachments (send and receive)
    - Text-based approvals (no inline buttons)
    - Proactive notifications from scheduled tasks
  </Tab>

  <Tab title="Lark">
    <Steps>
      <Step title="Create a bot application">
        Log in to the [Lark developer console](https://open.feishu.cn/app). Create a new app, enable the Bot capability, and set permissions (send messages, receive messages, upload files). Note the **App ID** and **App Secret**.
      </Step>
      <Step title="Configure event subscription">
        Set the request URL to Headmaster's webhook endpoint. Subscribe to the `im.message.receive_v1` event.
      </Step>
      <Step title="Connect in Headmaster">
        Go to **Channels → Lark → Connect**. Enter the App ID and App Secret. Save.
      </Step>
    </Steps>

    **What works:**
    - Text messages (send and receive)
    - File attachments (send and receive)
    - Inline buttons for approvals (Lark supports interactive cards)
    - Proactive notifications from scheduled tasks
  </Tab>

  <Tab title="WhatsApp Business">
    <Steps>
      <Step title="Sign up for Cloud API">
        Sign up at [Meta for Developers](https://developers.facebook.com/apps/). Add the WhatsApp product to your Meta app. Get the **Phone Number ID** and **Access Token**. Configure the webhook: set the callback URL to Headmaster's webhook endpoint and subscribe to message events.
      </Step>
      <Step title="Connect in Headmaster">
        Go to **Channels → WhatsApp Business → Connect**. Enter the Phone Number ID and Access Token. Save.
      </Step>
    </Steps>

    **What works:**
    - Text messages (send and receive)
    - Media messages (images, documents — limited by WhatsApp's API)
    - Inline buttons for approvals (WhatsApp supports interactive message buttons)
    - Proactive notifications from scheduled tasks
  </Tab>

  <Tab title="iMessage">
    <Note>
      iMessage is an advanced setup. Apple Messages doesn't have a public API — a Mac running as a bridge is required.
    </Note>

    <Steps>
      <Step title="Install the bridge">
        On your Mac, install the bridge software (follow the platform-specific guide for the latest compatible bridge). Configure the bridge to forward messages to Headmaster's webhook endpoint.
      </Step>
      <Step title="Connect in Headmaster">
        Go to **Channels → iMessage → Connect**. Enter the bridge endpoint URL and authentication token. Save.
      </Step>
    </Steps>

    **What works:**
    - Text messages only (no file transfer)
    - Text-based approvals
    - Proactive notifications

    **The bridge must stay online for the channel to work.**
  </Tab>
</Tabs>

## What flows over channels

- **Inbound** — messages from the platform become new turns in your active conversation.
- **Outbound** — the agent's responses are sent back to the platform.
- **File attachments** — supported on channels that allow them.
- **Approvals and clarifications** — work the same as in the desktop chat. On platforms without inline buttons, approvals are sent as text prompts.
- **Notifications** — the agent proactively messages you when a background task finishes or a scheduled task triggers.

---

## Proactive behaviors

The agent can reach out to you on channels without you asking:

- **Scheduled task results** — when a cron job fires, the result is delivered to your configured channel.
- **Background task completion** — when a long-running task finishes.
- **Approval requests** — if the agent needs approval and you're not at the desktop, it sends the request to your channel.
- **Reminders** — the agent can send scheduled reminders.

This is what makes Headmaster feel like a 24/7 assistant — it reaches out to you on the platform you're already on.

---

## Disconnecting

Click **Disconnect** on a channel card. The agent stops reading from and replying to that platform immediately. Past conversation transcripts are kept.
